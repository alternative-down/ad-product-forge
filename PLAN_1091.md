# Issue #1091: Todo Operacional — Implementation Plan

## Goal
Add `updateTodos` runtime action to agent runtime with native state persistence.

## Key Design Decisions

### Store
- New: `packages/forge-runtime-core/src/libsql-todo-store.ts`
- Inherits from pattern of `LibsqlConversationStore` (Client, tablePrefix)
- Table: `todos` with columns: id, thread_id, resource_id, title, status, created_at, updated_at
- Separate from conversation store, operational memory, Plan Mode, LTM

### Runtime Action: `updateTodos`
- Input schema (Zod): `{ items: TodoItemInput | TodoItemInput[] }`
- `TodoItemInput`: `{ id?: string; title: string; status?: 'pending' | 'in_progress' | 'completed' }`
- Rules from issue:
  - Object → create/update single
  - Array → create/update multiple
  - `items: []` → clear all
  - No id → create new (UUID)
  - Has id → update existing
  - No status on create → defaults to `pending`
  - No status on update → preserve existing
  - Atomic batch updates

### Integration Points
1. `packages/forge-runtime-core/src/runtime-agent-session.ts` — add `todoStore` and `runtimeActions` options
2. `packages/forge-runtime-core/src/runtime-agent-session-generate.ts` — pass actions to generate
3. `packages/forge-runtime-core/src/runtime-agent-session-runtime.ts` — inject `updateTodos` action via `getRuntimeActions()`
4. Context injection: todos rendered as short state block (not a tool call result)
